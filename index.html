How Data Augmentation Affects Deep Learning Performance

Abstract:¶

This tutorial will explore the impacts of data augmentation in relation to image classification. By training a Convolutional Neural Network (CNN) on the CIFAR-10 dataset and analysing how it compares to a baseline model and a model which uses an augmented dataset. Augmentation techniques such as flipping, jittering and rotating will be investigated, showing how they can be performed and what effects they have on an image. Readers should already have a basic understanding of CNNs. After completing the tutorial, they should gain a deeper understanding of data augmentation, its applications, and its impact on deep learning models.

Learning Objectives:¶

  • Understand how data augmentation impacts deep learning models
  • Check the data for any missing or invalid data in a dataset
  • Implement various augmentation techniques to images and see their effects
  • Construct a new dataset by augmenting data
  • Compare CNN training and validation on models using different datasets
  • Understand how to analyse results using various visual methods

# Function for checking NaN values in the dataset
def Data_Check(dataset):
    print("\nNumber of samples in dataset:", len(dataset))

    NaN_count = 0
    label_count = 0

    # Loop through the dataset
    for i in range(len(dataset)):
        image, label = dataset[i] 

        # Check for NaN values
        if torch.isnan(image).any():
            NaN_count += 1
            
        # Check if the label is not an integer
        if not isinstance(label, int):
            label_count += 1

    # Print out the results
    print("\nNumber of samples containing NaN values in images:", NaN_count)
    print("Number of samples containing NaN values in labels:", label_count)

Before creating the augmented dataset, let's visualise how each augmentation method affects an image. Each method described below will be individually applied to an image creating 6 more versions.
  1. Horizontal Flip: Flips image along the Y-axis
  2. Vertical Flip: Flips image along the X-axis
  3. Rotation: Rotates the image to a degree of choice
  4. Affine Translation: Moves the image in a random direction
  5. Colour Jitter: Randomly changes the brightness, contrast, saturation and hue
  6. Random Crop: Randomly crops a section of the photo with padding

def imshow_grid(images):

Question: What are some advantages and disadvantages of data augmentation?¶

Click to see the answer!

Advantages:

  • Improved generalisation - Model learns robust patterns
  • Reduced Overfitting - Model learns on more diverse data
  • Increased Accuracy - More data leads to increased accuracy
  • Cost Effective - Cheaper and faster than collecting new data

Disadvantages:

  • Increased computational cost - Augmenting the data takes computational power
  • May introduce noisy data - Data that is too augmented can lead to unrealistic distortions
  • Only effective in certain datasets - i.e. Vertical flipping in a sign language data set could alter the meaning of the image

Increasing the batch size can impact the model in the following ways:

Pros:

  • Faster Training - Larger batches require fewer updates to weights and biases
  • Lower Computational Costs - Larger batches require fewer updates to weights and biases

Cons:

  • Weaker Generalisation - Models more frequently get stuck in local minima
  • Slower Convergence - Less frequent updates can slow down learning

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 6, 5) # convolutional layer
        self.pool = nn.MaxPool2d(2, 2) # pooling layer
        self.conv2 = nn.Conv2d(6, 16, 5) # convolutional layer
        self.fc1 = nn.Linear(16 * 5 * 5, 120) # fully connected layer
        self.fc2 = nn.Linear(120, 84) # fully connected layer
        self.fc3 = nn.Linear(84, 10) # fully connected layer

    def forward(self, x):
        x = self.pool(F.relu(self.conv1(x))) # convolutional layer
        x = self.pool(F.relu(self.conv2(x))) # convolutional layer
        x = torch.flatten(x, 1) # flatten all dimensions except batch
        x = F.relu(self.fc1(x)) # fully connected layer
        x = F.relu(self.fc2(x)) # fully connected layer
        x = self.fc3(x) # fully connected layer
        return x

def train_model(criterion, optimizer, train_loader, val_loader):
    individual_loss = [] # Stores training losses after each 1000 images
    individual_val_loss = [] # Stores validation losses after each 1000 images

    for epoch in range(2):  # Loop over the dataset multiple times
        running_loss = 0.0

        # Training
        net.train()
        for i, data in enumerate(train_loader, 0):
            # get the inputs; data is a list of [inputs, labels]
            inputs, labels = data

            # Zero the parameter gradients
            optimizer.zero_grad()

            # Forward + backward + optimize
            outputs = net(inputs)
            loss = criterion(outputs, labels)
            loss.backward()
            optimizer.step()

            # Statistics
            running_loss += loss.item()
            if i %  evaluation == (evaluation-1): # Print and store stats every 1000 images
                r_loss = running_loss / evaluation
                print(f'[{epoch + 1}, {i + 1:5d}] loss: {r_loss:.3f}')
                individual_loss.append(r_loss)
                running_loss = 0.0

                # Validation
                net.eval()
                val_running_loss = 0.0
                with torch.no_grad():
                    for data in val_loader:
                        inputs, labels = data
                        outputs = net(inputs)
                        loss = criterion(outputs, labels)
                        val_running_loss += loss.item()

                val_loss = val_running_loss / len(val_loader)
                print(f'Validation Loss: {val_loss:.3f}')
                individual_val_loss.append(val_loss)
                # Change back to training mode after validation
                net.train()
                
    print('Finished Training')

    # Append individual losses to the overall list
    loss_values.append(individual_loss)
    val_loss_values.append(individual_val_loss)

# Stores the accuracys of the models
accuracy_totals = []

def accuracy(net, testloader):
    correct = 0
    total = 0
    # since we're not training, we don't need to calculate the gradients for our outputs
    with torch.no_grad():
        for data in testloader:
            images, labels = data
            # calculate outputs by running images through the network
            outputs = net(images)
            # the class with the highest energy is what we choose as prediction
            _, predicted = torch.max(outputs, 1)
            total += labels.size(0)
            correct += (predicted == labels).sum().item()

    tot_acc = 100 * correct // total
    print(f'Accuracy of the network on the 10000 test images: {tot_acc} %')
    print('Total Accuracy Complete!')
    accuracy_totals.append(tot_acc)

def class_accuracy(net, testloader):
    all_labels_save = []
    all_preds_save = []
    # prepare to count predictions for each class
    correct_pred = {classname: 0 for classname in classes}
    total_pred = {classname: 0 for classname in classes}

    # again no gradients needed
    with torch.no_grad():
        for data in testloader:
            images, labels = data
            outputs = net(images)
            _, predictions = torch.max(outputs, 1)

            # Append actual and predicted labels
            all_labels_save.append(labels)  
            all_preds_save.append(predictions)

            # collect the correct predictions for each class
            for label, prediction in zip(labels, predictions):
                if label == prediction:
                    correct_pred[classes[label]] += 1
                total_pred[classes[label]] += 1

    all_labels.append(all_labels_save)
    all_preds.append(all_preds_save)
    class_individual = []
    # print accuracy for each class
    for classname, correct_count in correct_pred.items():
        accuracy = 100 * float(correct_count) / total_pred[classname]
        class_individual.append(accuracy)
        print(f'Accuracy for class: {classname:5s} is {accuracy:.1f} %')
    classes_accuracy.append(class_individual)
    print('Class Accuracy Complete!')

What do these metrics mean?¶

If you are unsure click to see an explanation!

Precision: Out of all the positive predictions, how many are actually positive?

Recall (Sensitivity): Out of all actual positives, how many are predicted correctly?

F1-score: The harmonic mean of precision and recall, a single metric that balances both metrics above.

Support: Count of samples in the given classes

Conclusion¶

This tutorial showcases a machine learning pipeline using PyTorch within Jupiter notebooks. From downloading the data to creating captivating visuals, it displays the essential steps required in a pipeline, while also emphasising the role of data augmentation can have.

Throughout this tutorial, we have discovered that when limited data is available, data augmentation can be used as a feasible option. Although new and different data is the most effective option, augmentation boasts impressive results as a quick, simple, and effective way to expand the size of the dataset. In our example, it increased the overall accuracy from baseline: 45% to augmented: 50%, showcasing its potential. This is further supported by the bar chart, accuracy metrics and confusion matrices which visualise a more in-depth comparison between all three datasets.

However, in specific classes, augmentation did not improve the baseline model and potentially introduced noise into the dataset. As has been mentioned throughout, the model itself and the augmented data can be further fine-tuned to increase accuracy.

Future work would expand on this. A deeper analysis of each data augmentation technique could help identify the most optimal transformations, rather than randomly assigning them. In addition, experiments involving more extreme augmentation methods could be conducted to see whether they positively impact the model or introduce more noise in the dataset.
